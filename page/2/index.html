SharedPreferences

发表于 2017-11-01 | 更新于 2019-04-10 | 分类于 Android基础

简介

SharedPreferences是一种轻型的数据存储方式，本质是基于XML文件存储key-value键值对数据。支持多种数据类型存储。通常用来存储一些简单的配置信息，常用于程序的偏好设置功能和记住密码功能。

将数据存储到SharedPreferences中(data/data/包名/shared_prefs中)

  1. 获取SharedPreferences对象

    • 通过Context 类中的getSharedPreferences()方法获取。

- 通过_Activity_类中的`getPreferences()`方法获取。使用这个方法时会将当前活动的类名作为SharedPreferences的文件名。

从SharedPreferences中读取数据

SharedPreferences对象中提供了了一系列getxxx()方法用于对存储的数据进行读取，每种get方法都对应了SharedPreferences.Editor对象的一种putxxx()方法。getxxx()的第一个参数为键，第二个参数为默认值。示例代码

#4. Socket通信简介

  • TCP协议是面向连接、可靠的、有序的，以字节流的方式发送数据。
  • 基于TCP协议实现网络通信的类：客户端的Socket类和服务端的ServerSocket类。
  • Socket通信模型：
  • Socket通信实现步骤：
    1. 在服务端创建ServerSocket，在客户端创建Socket。
    2. 打开连接到Socket的输入/输出流
    3. 按照协议对Socket进行读/写操作
    4. 关闭输入/输出流和Socket

#5. 基于TCP的Socket通信(先启动服务端，再启动客户端)

  • 服务器端：
    1. 创建ServerSocket对象，绑定监听端口。
    2. 通过accept()方法监听客户端请求。
    3. 连接建立后，通过输入流读取客户端发送的请求消息。
    4. 通过输出流向客户端发送响应信息。
    5. 关闭相关资源。
  • 客户端：
    1. 创建Socket对象，指明需要连接的服务器的地址和端口号。
    2. 连接创立后，通过输出流向服务器端发送请求信息。
    3. 通过输入流获取服务器响应的信息
    4. 关闭相关资源。
  • 多线程服务器：应用多线程来实现服务器月多客户端之间的通信
    1. 服务器端创建ServerSocket，循环调用accept()等待客户端连接
    2. 客户端创建一个Socket并请求和服务器端连接
    3. 服务器端接收客户端请求，创建socket与该客户建立专线连接
    4. 建立连接的两个Socket在一个单独的线程上对话
    5. 服务器机继续等待新的连接

#6. UDP编程

  1. UDP协议(用户数据报协议)是无连接、不可靠、无序的。
  2. 以数据报作为传输的载体：以UDP最为数据传输时，首先需要将要传输的数据定义成数据报(Datagram)，在数据报中指明数据将要达到的Socket，然后再将数据报发送出去。
  3. 相关操作类：DatagramPacket：表示数据报包；DatagramSocket：进行端到端通信的类。
  • 服务端：
  1. 创建DatagramSocket,指定端口号
  2. 创建DatagramPacket
  3. 接收客户端发送的数据信息
  4. 读取数据
  • 客户端：
  1. 定义服务器的地址、端口号和要发送的信息
  2. 创建数据报(DatagramPacket)，包含将要发送的信息
  3. 创建DatagramSocket，实现数据的发送
  4. 发送数据
